# 🌐 Burp Suite

Burp Suite is an **industry-grade tool** for hands-on **security assessments** of web applications and APIs.  

- Captures and manipulates all **HTTP/HTTPS traffic** between browser ↔️ server.  
- Lets you **modify, replay, analyze, and attack** requests/responses.  
- Versions available:  
  - **Community Edition** (Free, limited features)  
  - **Professional Edition** (Full features for pentesters)  
  - **Enterprise Edition** (Automation, CI/CD testing)  

---

## 🛠️ Features of Burp Community

### 🔹 Proxy
- Acts as an **intercepting proxy** between browser ↔️ web server.  
- Lets you **pause, view, and modify requests** before they hit the server.  

**Example:**  
Intercepted request before login form submission:  
```http
POST /login HTTP/1.1
Host: vulnerable-website.com
Content-Type: application/x-www-form-urlencoded

username=admin&password=admin123
````

Modified request (SQL Injection attempt):

```http
username=admin' OR '1'='1&password=anything
```

---

### 🔹 Repeater

- Allows capturing, modifying, and **resending requests repeatedly**.
    
- Useful for testing payloads (e.g., SQLi, XSS).
    

**Example:**

```http
GET /product?id=1 HTTP/1.1
Host: vulnerable-website.com
```

→ Modify `id=1` to `id=1 OR 1=1` → resend and analyze.

---

### 🔹 Intruder

- Automates **brute-force attacks** and **fuzzing**.
    
- Community Edition has **rate limits** (slower).
    

**Example:** Brute force login with wordlist:

```http
POST /login HTTP/1.1
username=admin&password=§password§
```

(`§ §` marks attack payload positions)

---

### 🔹 Decoder

- Used to **encode/decode** or **transform data** (URL, Base64, Hex, HTML).
    

**Example:**

- Input: `admin:1234`
    
- Encoded (Base64): `YWRtaW46MTIzNA==`
    

---

### 🔹 Comparer

- Compare two sets of data at **byte-level or word-level**.
    
- Useful for checking small differences in responses (e.g., success vs failure).
    

---

### 🔹 Sequencer

- Tests **randomness of tokens** (e.g., session cookies, CSRF tokens).
    
- Helps determine predictability of authentication mechanisms.

## ⌨️ Shortcuts (Navigation)

- **Ctrl + Shift + D** → Dashboard
    
- **Ctrl + Shift + T** → Target Tab
    
- **Ctrl + Shift + P** → Proxy Tab
    
- **Ctrl + Shift + I** → Intruder Tab
    
- **Ctrl + Shift + R** → Repeater Tab
    

---

## ⚙️ Options & Tips

- When modifying requests, **always URL encode payloads** using:
    
    - **Ctrl + U** → URL encode selected text
        

**Example:**

- Normal: `<script>alert(1)</script>`
    
- URL Encoded: `%3Cscript%3Ealert(1)%3C%2Fscript%3E`
